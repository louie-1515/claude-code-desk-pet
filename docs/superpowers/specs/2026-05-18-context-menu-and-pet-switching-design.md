# Claude 桌宠右键菜单、待机隐藏气泡与形象切换设计

日期：2026-05-18

## 目标

在不破坏现有拖动、缩放、状态桥接和托盘行为的前提下，为 Claude 桌宠补上三项能力：

1. 直接右键桌宠时，弹出与托盘右键一致的菜单功能。
2. 支持“仅在待机时隐藏气泡”的可切换设置，默认关闭。
3. 支持在菜单中切换多种内置桌宠形象，并为后续扩展成目录扫描保留清晰边界。

## 非目标

这次不做下面这些内容：

1. 不实现自定义文件夹自动扫描形象。
2. 不在菜单里做复杂的角色预览、缩略图或编辑器。
3. 不改动 Claude Hook、状态映射、拖动语义和缩放交互本身。
4. 不引入新的外部依赖。

## 用户体验

### 1. 右键桌宠

用户在桌宠窗口上右键时，看到与托盘右键同一套菜单能力。菜单中至少包含：

1. 显示/隐藏桌宠
2. 重载桌宠
3. 恢复默认位置
4. 打开项目目录
5. 打开状态文件
6. 待机时隐藏气泡（复选开关）
7. 切换形象（子菜单）
8. 退出

右键菜单与托盘菜单共享同一个菜单模型，避免功能漂移。

### 2. 待机隐藏气泡

默认行为保持现在的样子：气泡显示。

当“待机时隐藏气泡”开启后：

1. `idle` 阶段隐藏气泡。
2. `thinking`、`tool_running`、`waiting_input`、`needs_approval`、`done`、`error` 等非待机阶段仍正常显示气泡。
3. 拖动过程中仍显示拖动文案，不受该开关影响。

这样既保留了安静待机时的简洁感，也不会把用户真正需要看的状态提示藏掉。

### 3. 切换形象

右键菜单中提供“切换形象”子菜单，列出内置形象。

本期先使用内置注册表模式：

1. 菜单中列出内置形象名称。
2. 当前选中形象以单选勾选形式显示。
3. 切换后立即持久化，并刷新当前桌宠窗口。
4. 下次启动时自动恢复上次选择。

## 架构设计

### 1. 设置存储

当前项目已经有 `bridge/state/window-state.json`，里面存放位置与 `petScale`。这次沿用该文件，但明确区分其职责为“窗口与展示偏好状态”。

新增字段：

1. `selectedPetId: string`
2. `hideBubbleOnIdle: boolean`

保留现有字段：

1. `x`
2. `y`
3. `petScale`

统一通过现有的读取、归一化、写回链路持久化，避免再造第二份零散设置文件。

### 2. 内置形象注册表

新增一个主进程可读、renderer 可消费的内置形象注册表模块，例如：

1. `id`
2. `name`
3. `petConfigFile`
4. `spriteFile`
5. `trayIconFile`（可选）

当前 `assets/pet.json` 和 `assets/spritesheet.png` 不再被写死成唯一来源；主进程启动时根据 `selectedPetId` 解析当前激活形象。

本期只登记内置形象。后续扩展成扫描 `pets/` 目录时，这个注册表模块可以演进为“内置 + 目录发现”的聚合层，而不用改菜单和窗口装配逻辑。

### 3. 菜单模型

将托盘菜单构造逻辑抽成一个共享方法，例如 `buildDeskPetMenuTemplate()`。

输入：

1. 当前是否有主窗口
2. 当前设置
3. 当前形象列表
4. 当前选中形象

输出：

1. Electron `Menu` 所需模板

托盘菜单与桌宠右键菜单都从这一个模板生成。这样新增菜单项只维护一处。

### 4. 右键菜单触发

Renderer 在桌宠窗口中监听右键事件，通知主进程显示上下文菜单。

要求：

1. 右键不会触发拖动。
2. 右键不会误触发左键单击聚焦 Claude 终端。
3. 菜单显示位置跟随当前鼠标位置。

### 5. 待机气泡显示规则

Renderer 渲染时引入一个单独的“气泡可见性判断”，不要把它硬塞进动画状态机。

建议规则：

1. 如果当前是拖动态，显示气泡。
2. 如果 `hideBubbleOnIdle === true` 且 `phase === "idle"`，隐藏气泡。
3. 否则显示气泡。

这样显示逻辑与动画逻辑分离，后续更容易维护。

### 6. 切换形象后的窗口刷新

切换形象后，主进程需要让窗口重新读取新的资源路径。最稳妥的本期方案是：

1. 持久化新的 `selectedPetId`
2. 更新托盘 tooltip / icon（如果该形象定义了独立图标）
3. 重新加载窗口，让 preload 与 renderer 重新获取 boot payload

不建议在本期做“只替换局部资源、不重载窗口”的热切换，因为那会让资源状态、缩放状态和渲染生命周期耦合得更深。

## 数据流

### 启动

1. 主进程读取 `window-state.json`
2. 解析 `selectedPetId`
3. 从内置注册表中解析当前形象资源
4. 创建窗口时把当前形象资源路径传给 preload / renderer
5. Renderer 读取 `hideBubbleOnIdle` 和当前形象配置，完成启动渲染

### 右键菜单

1. 用户在桌宠窗口右键
2. Renderer 发 IPC 给主进程
3. 主进程基于共享菜单模板弹出上下文菜单
4. 用户点击某项后，主进程执行对应动作

### 切换待机气泡开关

1. 用户点击菜单复选项
2. 主进程更新 `window-state.json`
3. 主进程通知 renderer 刷新设置，或者直接重载窗口
4. Renderer 按新设置更新气泡显示

本期更推荐主进程把最新设置通过 IPC 主动推给 renderer，而不是每次都整窗重载。因为这只是显示偏好变化，没必要重启整只桌宠。

### 切换形象

1. 用户点击某个形象单选项
2. 主进程更新 `selectedPetId`
3. 主进程更新托盘表现
4. 主进程重载窗口
5. Renderer 通过新的 boot payload 读取新角色配置与精灵图

## 错误处理

### 1. 选中的形象不存在

如果 `selectedPetId` 在注册表中找不到：

1. 自动回退到默认形象
2. 修正持久化状态为默认形象
3. 在控制台输出清楚的 warning

### 2. 形象资源损坏

如果 `pet.json` 或 `spritesheet.png` 不可用：

1. 尝试回退到默认形象
2. 如果默认形象也不可用，保留当前已有的 renderer 降级逻辑，不让窗口直接白屏崩掉

### 3. 设置文件异常

如果 `window-state.json` 解析失败：

1. 沿用现有默认窗口状态
2. `hideBubbleOnIdle` 默认 `false`
3. `selectedPetId` 默认默认形象

## 测试策略

### 单元测试

1. 设置归一化测试：
   - `hideBubbleOnIdle` 默认值
   - `selectedPetId` 默认值与非法值回退
2. 形象注册表测试：
   - 默认形象存在
   - 注册项资源路径正确可解析
3. 菜单模板测试：
   - 右键/托盘共用模板
   - 待机隐藏气泡是复选项
   - 切换形象是单选子菜单
4. Renderer 测试：
   - `idle + hideBubbleOnIdle=true` 时隐藏气泡
   - `done/error/waiting/dragging` 时仍显示气泡

### 手动验证

1. 右键桌宠可以弹菜单
2. 菜单项与托盘一致
3. 勾选/取消“待机时隐藏气泡”立即生效
4. 切到其他形象后资源正确刷新
5. 重启应用后设置仍然保留
6. 不影响左键拖动、左键点击聚焦、双击打开项目目录、右下角缩放

## 实现拆分与模型分工

这次适合并行，按复杂度拆成三块：

### Agent A：`gpt-5.2`

范围：

1. 设置状态归一化与持久化扩展
2. 菜单模板抽取与测试

理由：

这是边界清晰、规则明确的多文件小改，适合便宜模型。

### Agent B：默认主力模型

范围：

1. 主进程右键菜单触发链路
2. 当前形象注册表与资源切换链路
3. 切换后重载窗口与托盘同步

理由：

这里涉及主进程、窗口生命周期、托盘、资源装配，多模块耦合更高，需要更稳的判断。

### Agent C：`gpt-5.2`

范围：

1. Renderer 的待机气泡隐藏逻辑
2. 相关单元测试或轻量渲染测试

理由：

这是明确的显示逻辑改动，边界很清楚，适合低成本模型并行处理。

主智能体负责：

1. 统筹接口边界
2. 整合冲突
3. 全量回归验证
4. 最终清理与验收

## 风险与取舍

1. 如果把“切换形象”直接做成完全热更新，复杂度会明显升高；本期选择切换后重载窗口，优先稳。
2. 如果把右键菜单单独维护一套，很快会和托盘菜单漂移；本期明确只保留一份菜单模型。
3. 如果把“隐藏气泡”做成全局隐藏，会把真正重要状态提示也藏掉；本期只隐藏待机态。

## 结论

本期采用以下方案：

1. 共享菜单模板，桌宠右键和托盘右键同源
2. `hideBubbleOnIdle` 持久化设置，默认关闭
3. 内置形象注册表 + 菜单子菜单切换
4. 切换形象后重载窗口，保证实现稳妥

这套设计能一次解决你现在要的三个能力，并且自然为下一步“扫描 `pets/` 目录自动发现形象”留出扩展位置。
